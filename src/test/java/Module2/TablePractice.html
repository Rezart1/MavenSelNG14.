-------------------------------------------------------------------------- - BIG IDEA: There can be more tricky web elemnst that are harder to locate. It can be due to many reasons, like absence of attributes, complex layouts, generic tab names etc... It get's even more tricky when the web elements are dynamic that are changing based on user decisions/actions. We need a reliable strategy how to deal with those elements. Strategies to use: 1. Using text() function can help to identify an element by the content of it. ex:
Hello
-> //div[text() = 'Hello'] 2. Using contains() function that can match a part of the value we are reffring to. We can use the values of the attributes or even text of the element. The contains() function will take two arguments -> contains(@attribute, 'value') or contains(text(), 'value') 3. Using and - or logical operators to combine criterial for better filtering //input[@name = 'sampleInput' and @class = 'differentClass'] //input[@type = 'username' or @type = 'password'] 4. Using starts-with() function to get elemnts whereas some attributes or text have a specific prefix. //input[starts-with(@id, 'user')] -------------------------------------------------------------------------- -------------------------------------------------------------------------- BIG IDEA: Web Tables are needed for organizing the information in table like format In html when used good semantics, a table with be created with following tags:
